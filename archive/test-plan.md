## Conventions

Each E2E is a bash script under `test/` (or whatever Stream C settles on).
Setup creates a temp `~/.arianna/` and points `ARIANNA_HOME` at it so the
test never touches the operator's real config. Cleanup tears down all
docker containers/images/tags it created — no per-run residue.

Every script prints a single `PASS:` or `FAIL: <why>` line at the end.
Stream C's harness greps for these.

## E1 — install.sh first run (idempotent)

**Script:** `test/install-firstrun.sh`

**Setup:** Fresh tmp `ARIANNA_HOME`. Mock `npm install -g` (no real network)
or use a verdaccio-like local registry. Real Docker daemon required.

**Steps:**
1. `bash install.sh` from a fresh checkout. Confirm exit code 0.
2. Confirm `~/.arianna/repo/` exists and contains `docker-compose.yml`.
3. Confirm `~/.arianna/config` is **empty** (legacy single-tenant flow,
   no profile auto-registered).
4. Confirm `arianna --version` works from any cwd.
5. Confirm `arianna profile current` prints
   `default\t(source: implicit-default)`.
6. Run `bash install.sh` again. Confirm exit code 0, `git pull --ff-only`
   succeeds, no duplicate clones, no docker rebuild churn beyond what
   the daemon itself reports.

**Pass:** All steps succeed. Failures on docker/node missing produce the
documented platform-specific error message.

## E2 — CLI happy path (talk + events)

**Script:** `test/cli-lifecycle.sh`

**Setup:** install.sh has run. Vessel + sidecar built but not yet running.

**Steps:**
1. `docker compose -f ~/.arianna/repo/docker-compose.yml up -d` —
   bring up the legacy single-tenant stack.
2. Wait for `arianna events` to print at least one `memory_state` event.
3. `arianna talk "hello, who are you?"` — confirm a non-empty response is
   streamed to stdout, exit code 0.
4. `arianna events --follow &` — capture for 10s, kill, confirm at least
   one `memory_state` and one `bookmark_snapshot` event were emitted.
5. Tear down: `docker compose down`, remove the session image.

**Pass:** All HTTP calls succeed, SSE drains correctly, vessel reports
healthy throughout.

## E3 — Profile lifecycle (create / use / talk / delete)

**Script:** `test/cli-lifecycle.sh` (extended)

**Setup:** install.sh has run; Docker available; no other arianna stack
running.

**Steps:**
1. `arianna profile create alpha` — confirm
   `workspace/profiles/alpha/compose.override.yml` exists, port_offset=0
   (if 3000 free) or next free offset.
2. `arianna profile list` — confirm alpha row, "*" default marker.
3. Bring up alpha's stack:
   `docker compose -p arianna-alpha -f docker-compose.yml \
    -f workspace/profiles/alpha/compose.override.yml up -d`.
4. `arianna --profile alpha talk "hi"` — confirm response.
5. `arianna --profile alpha events --follow &` — capture and verify.
6. Tear down alpha's stack via the same compose command + `down`.
7. Manually remove `~/.arianna/config` entry and
   `workspace/profiles/alpha/` (no `arianna profile delete` yet — that's
   a follow-up task).

**Pass:** Profile-aware vessel/sidecar reachable on shifted ports;
daemon-side ops route to alpha's `workspace/profiles/alpha/...` paths.

## E4 — Fork preserves state, source untouched

**Script:** `test/cli-fork.sh`

**Setup:** Profile alpha exists with at least one snapshot taken (talk
to it once so /sync triggers a daemon /snapshot). Docker available.

**Steps:**
1. Capture src state: `cp -r workspace/profiles/alpha /tmp/alpha-snapshot`.
   Note alpha's sessionId from `session_config.json`.
2. `arianna fork alpha beta` — confirm exit 0 and the summary prints
   "retagged N docker images".
3. Verify dst state:
   - `workspace/profiles/beta/session_config.json` exists with a NEW
     sessionId (different from alpha's) and same model/aiName/aiUsername.
   - `workspace/profiles/beta/compose.override.yml` exists with shifted
     ports.
   - `workspace/profiles/beta/sidecar-state/sessions/{beta-sessionId}.json`
     exists (file copied from alpha's session_state).
   - `workspace/profiles/beta/sidecar-state/snapshot-histories/snap_*.json`
     each have `sessionId` set to the new beta sessionId.
   - `docker images --filter reference=ariannarun-vessel:{beta-sessionId}-*`
     returns at least `-base` and `-current`.
4. Verify src untouched:
   - `diff -r /tmp/alpha-snapshot workspace/profiles/alpha` exits 0
     (byte-equal).
   - `docker images --filter reference=ariannarun-vessel:{alpha-sessionId}-*`
     still returns the same set as before fork (no rmi).
5. Bring up alpha's stack again, confirm it still talks and shows its
   original state via `/map` snapshots.
6. Bring up beta's stack on the shifted ports, confirm independent state.

**Pass:** Both profiles run independently, sourced from the same docker
base image but never sharing mutable state.

## E5 — Default-profile sentinel guard

**Script:** `test/cli-default-guard.sh`

**Setup:** Inside the canonical repo (`~/.arianna/repo` or a checkout).
`~/.arianna/config` empty.

**Steps:**
1. `mkdir -p workspace/profiles/default && touch workspace/profiles/default/.no-default-allowed`.
2. `arianna talk "hi"` (no `--profile` flag, no `ARIANNA_PROFILE` env).
   Confirm exit code != 0 with a message containing "implicit default".
3. `arianna profile current` — confirm message containing "no profile
   resolved" and the sentinel path.
4. With `ARIANNA_PROFILE=alpha` (alpha not registered) — `arianna talk`
   itself does NOT trigger the daemon's unknown-profile 404 because the
   talk command posts to the vessel directly; it just lands on whatever
   port `lookupOffset` resolves (silently 0 for unknown profiles). Use a
   daemon-bound endpoint to exercise the unknown-profile path:
   `curl -sS -w "[%{http_code}]" 'http://127.0.0.1:9000/diff?profile=alpha'`
   should return `[404]` with body `{"code":"unknown-profile",...}`.
5. Set up an actual default in config: `arianna profile create alpha;
   arianna profile use alpha`. Confirm `arianna talk` now resolves to
   alpha (sentinel is bypassed because we have an explicit
   config-default — sentinel only blocks IMPLICIT default).
6. Cleanup: remove `.no-default-allowed`, remove alpha.

**Pass:** Sentinel correctly blocks the literal-`default` fallback while
allowing explicit config-default to work.

## E6 — Parallel `arianna profile create` race

**Script:** `test/cli-port-race.sh`

**Setup:** Empty `~/.arianna/config`. Inside a repo with no existing
profiles. Docker available (or use `--skip-bind-test`-equivalent env if
exposed for tests).

**Steps:**
1. Spawn 5 `arianna profile create p1 p2 p3 p4 p5` invocations in
   parallel. (Adapt to exactly 5 single-profile calls running concurrently
   across 5 shells — not literal multi-arg.)
2. Wait for all to finish.
3. Confirm exactly 5 entries in `~/.arianna/config`, each with a unique
   `port_offset` in [0..4].
4. Confirm `~/.arianna/ports.lock` is removed (no leaked lockfile).
5. Spawn 100 parallel creates. Confirm any beyond the
   bind-test-available port range fail with the documented
   "no free port offset" error.

**Pass:** No two profiles get the same offset. ports.lock is reclaimed.

## E7 — Stream B / OpenClaw smoke against the new daemon contract

**Script:** Stream B owns this. From Stream A's perspective, the contract
to verify is:

- `GET /health` returns 200 without a profile param.
- All other endpoints accept either no profile (→ legacy fallback) OR
  `?profile=name` OR `X-Arianna-Profile: name` header.
- `?profile=invalid+name` → 400.
- `?profile=ghost` (valid format, not in `~/.arianna/config`) → 404.
- Both query and header set with conflicting values → 400.
- `ARIANNA_DAEMON_STRICT=1` env makes missing-profile → 400 instead of
  legacy fallback.

These are also covered as unit tests in
`packages/host/test/daemon-profile.test.ts` (no Docker needed) — the
E2E version is to confirm the full request stack agrees.

## What's deliberately NOT in this plan

- **`arianna profile delete`** — not implemented in Stream A's scope.
  When implemented, it owns its own E2E that asserts compose down + dir
  removal + image cleanup.
- **OpenClaw imported partner flow** — Stream B/C territory. Stream A's
  `arianna profile create` doesn't yet take an `--import` flag.
- **Hint cadence threshold tuning** — Stream C's #30. Stream A only
  plumbs the `cadence` field through `SessionConfig` and the fork spread.
- **Vessel container restart mid-`arianna talk`** — covered by the
  existing host TUI's reconnect logic, which is unchanged. The CLI's
  `talk` command surfaces the disconnect as an error and exits non-zero;
  it doesn't currently auto-reconnect.
